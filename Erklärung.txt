OVERVIEW
--------
This folder contains all input data for the OPF optimization of the Swiss
low-voltage (LV) distribution grid 2703-23_0_4, located in Riehen (BFS
municipality code 2703, Midlands-Urban). Data is provided for three planning
horizons (2030, 2040, 2050) and two representative-day configurations.

Grid properties:
  - 178 nodes (buses), coordinates in CH1903+ / LV95
  - Scenario years: 2030, 2040, 2050
  - Technologies: PV, BESS, Heat Pumps (HP), Electric Vehicles (EV), Demand

================================================================================
FILE FORMATS & UNITS
================================================================================

rep_days_summary.csv
  - day:    date label MM-DD
  - index:  0-based day-of-year index
  - weight: number of real days this representative day stands for (sum = 365)
  Usage in OPF: multiply objective terms by weight to obtain annual values.

nodes.geojson / edges.geojson
  - Standard GeoJSON, CRS: EPSG:2056 (CH1903+ LV95)
  - nodes: osmid, x, y, el_dmd, Category, consumers, voltage
  - edges: line parameters (impedance, capacity)

load_profiles.csv / pv_generation.csv / pv_std.csv
  - Rows: one per node (identified by LV_grid, LV_osmid)
  - Columns: timestamps MM-DD HH:MM:SS for each hour of each rep. day
  - Units: kW

pv_p_installed.csv
  - Static: installed PV peak power per node [kW]

bess_allocation.csv
  - Battery_capacity_kWh:      energy capacity [kWh]
  - Nominal_power_kW:          max charge/discharge power [kW]
  - Charging_efficiency:       eta_c = 0.922 [-]
  - Discharging_efficiency:    eta_d = 0.922 [-]

temperature_profiles.csv
  - Rows: one per temperature profile type
  - Columns: hourly timestamps [degC]
  - Used to compute HP power consumption

ev_allocation.csv
  - EV_share per node: fraction of total municipality EV fleet at this node
  - Node EV load [kW] = EV_share × ev_power_profiles [kW]

ev_power_profiles.csv
  - Municipality-level total EV charging load [kW]
  - 3 rows: Lower / Base / Upper scenario
  - Columns: hourly timestamps

ev_flexible_energy_profiles.csv
  - Municipality-level daily flexible EV energy [kWh/day]
  - Constraint in OPF: sum of flexible charging per day <= this value
  - Node flexible energy [kWh] = EV_share × ev_flexible_energy_profiles [kWh]

================================================================================
USAGE NOTES FOR OPF
================================================================================

1. Representative days:
   Use rep_days_summary.csv to get the weight w_d for each day d.
   Annual objective = sum_d (w_d * cost_d)

2. Node-level EV load:
   P_EV_node(i, t) = ev_allocation[i] * ev_power_profiles[scenario, t]

3. BESS constraints:
   - Energy balance: SoC(t+1) = SoC(t) + eta_c*P_charge(t) - P_discharge(t)/eta_d
   - Power limits:   P_charge, P_discharge <= Nominal_power_kW
   - Energy limits:  0 <= SoC <= Battery_capacity_kWh

4. EV flexibility:
   sum_t(P_flex(i,t)) <= ev_allocation[i] * ev_flexible_energy_profiles[d]

5. Choosing between 2703_23 and 2703_23_homogen:
   - 2703_23 (20 days):        better statistical representation of annual variation
   - 2703_23_homogen (24 days): guaranteed seasonal balance, easier to interpret
